Power-user coin types. These are secondary to your everyday Tokens, Collectibles, and Trades — open one only if you know you need it. Each panel keeps the plain idea up front and the protocol detail behind a toggle.

Reversible payments (clawback)

Send funds that the recipient can only claim after a delay — and that you can take back until they do.

Claim or recover an existing one (protocol detail)

A clawback coin is a 1-of-2 puzzle: the receiver path (after the delay) and the sender recover path. Supply the coin's terms to claim (you're the receiver) or recover (you're the sender).

Options (covered calls)

Lock XCH as an option's underlying; the option holder can exercise it (pay the strike) before it expires, or you reclaim the underlying after.

Exercise / clawback of options needs a full Chia wallet in this build.

Streaming payments (vesting)

Pay XCH that unlocks gradually to a recipient between a start and end time. The recipient claims what's vested; you can claw back the rest.

Claim vested / claw back (protocol detail)

Reconstruct the live stream from the coin spend that created/updated it, then claim the vested portion (recipient) or claw back the remainder (payer). Needs a spare 0-mojo coin at your address to carry the authorizing message.

Vaults (multi-key custody)

Create a vault whose custody is an m-of-n set of secp256k1 (K1) keys — an independent multi-key custody set you manage alongside this wallet.

Spending a vault needs its K1 custody keys, which this wallet does not hold — create here, spend with the K1 keys elsewhere.

Verifiable credentials (attestations)

Compute the on-chain "asserter" address for a credential — the address a relying party sends a coin to so spending it requires proof the credential was issued.

Issuing / revoking a credential needs the issuing DID's tooling (it co-signs the same bundle) — not available standalone in this build.
